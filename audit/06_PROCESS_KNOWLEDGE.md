## Принцип блока

Меряем не «знают ли копирайтеры, где красивая кнопка», а **«где документировано знание о бренд-гайде, форматах, правилах, и что произойдёт, если носитель знаний недоступен»**.

## Что реально есть

### Всё знание — в git-репозитории как .md:

- `prompts/system_instructions.txt` — мета-инструкция
- `formats/email/SPEC_EMAIL_V1.md` + `formats/multiformat/SPEC_MULTIFORMAT_V1.md`
- `rules/SW_LEXICON_AND_BANS.md`
- `rules/SW_GLOBAL_RULES_v1.md`
- `validatorMultiformat.js` — лимиты и чёрные списки **в коде**

### Что входит в пилот по процессу:

- **RUNBOOK.md** — пошаговая инструкция «что делать, если упало», «как добавить формат», «как обновить лексикон»
- **Golden tests** — набор фиксированных брифов и эталонных результатов; CI прогоняет их при каждом изменении
- **Branch protection** на main — нельзя залить изменение в knowledge без code review
- **Версионирование** — git история, видно кто/когда/зачем изменил формулировку
- **Pull request template** — обязательное поле «какой формат это затрагивает» + «прогнал ли golden test»

### Bus factor:

- Сейчас: **2 человека** (Вера + разработчик)
- После пилота: **3+ человека** через RUNBOOK + golden tests
- Сравнение: у Custom GPT bus factor = 1 (владелец GPT) и нет передачи

## Вердикт по конкурентам

Все 7 — 🟢. Это **категорическая** разница: чат-продукты в принципе не проектировались под bus factor и под управление знаниями как кодом.

- **Custom GPT / Claude Projects**: можно загрузить файлы, но: нет git, нет диффа «что изменилось между версиями v3 и v4 бренд-гайда», нет ревьюера, нет CI.
- **ChatGPT Enterprise**: там есть «shared workspace», но это shared chat history, не version control над промптом.

## Сценарий-кейс для встречи

> «Вера 17 числа уходит в отпуск. Что происходит у нас и что происходит с Custom GPT?»

| Шаг | У нас (с RUNBOOK + git) | У Custom GPT |
|---|---|---|
| Бренд-команда хочет добавить формат «Reels-сценарий» | PR с новым `SPEC_REELS_V1.md` + правила в валидатор + golden test | Кто-то логинится под аккаунтом Веры (если знает пароль) |
| Изменение лексикона: «винотека» → «винный бутик» | Edit .md → PR → review → merge → CI прогоняет 47 тестов | Edit в чате Custom GPT, никто не знает, что изменилось |
| Что-то сломалось в продакшне | git blame + golden tests показывают, какая правка сломала | «Кто-то что-то поменял, давайте откатывать вручную» |
| Через год хотим вернуть как было в марте | git checkout | ❌ невозможно |

## Честные оговорки

- **Нужна культура PR** — копирайтеры должны привыкнуть к ревью своих правок. **Это не Word.** Часть пилота — обучение и онбординг (включено в стоимость).
- **GitHub доступ** — нужны учётки для команды. Не проблема, но IT надо предупредить.
- **Golden tests требуют поддержки** — при каждом новом формате надо добавлять тест. Это **процесс**, не магия. Включено в поддержку.

## Главный аргумент для презентации

> «Корп-знание SimpleWine о tone of voice, лексиконе, запретах — у нас в **git-репозитории**, не в голове Веры или в чужом облаке. Уйдёт Вера в отпуск, уйдёт из компании — RUNBOOK останется, golden tests упадут, если кто-то сломает — увидим до релиза. У Custom GPT всё это живёт в аккаунте создателя, и через год никто не вспомнит, почему было написано именно так.»

## Итог блока
