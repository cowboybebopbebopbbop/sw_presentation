## Принцип блока

Меряем не «можно ли скопировать ответ из чата», а **«может ли система быть встроена в реальный production-конвейер маркетинга: CMS → бриф → генерация → валидация → готовый артефакт → выгрузка в каналы»**.

Категориальная разница: **chat surface vs production runtime**. Chat-продукты архитектурно — окна для общения с моделью. Они **не предназначены** для встраивания в продакшн.

## Что уже работает E2E (это факты, не roadmap)

- ✅ **SEO-статья** — текст
- ✅ **Изображения через Gemini API** для статей (не stock, генерация)
- ✅ **Готовая HTML-вёрстка статьи** — на выход уходит не текст, а готовый к публикации HTML
- ✅ **Multiformat — 65 текстов одним прогоном** (13 форматов × ~5 вариантов)
- ✅ **DOCX-импорт брифа** через mammoth.js
- ✅ **MD-экспорт** через turndown
- ✅ **Reveal.js презентация результата** локально

То есть **минимум 4 интеграции** уже в продакшне (Anthropic + Gemini + mammoth + turndown), и пайплайн от брифа в DOCX до готового HTML-артефакта существует.

## Что архитектурно возможно (roadmap пилота, 1-2 недели каждое)

| Интеграция | Что делает | Сроки | Сложность |
|---|---|---|---|
| **Mindbox / UniSender / Sendsay** | Прямая выгрузка email-кампаний | 1-2 нед | средняя |
| **Push-система SimpleWine** | Готовый JSON в формате push-провайдера | 1 нед | низкая |
| **SMS-провайдеры** (SMSC.ru и т.п.) | Прямая отправка sms-вариантов на тест | 3 дня | низкая |
| **Bitrix / 1С / headless CMS** | Публикация SEO-статьи в каталог | 2 нед | средняя |
| **PIM / SKU-интеграция** (запрос Алины) | Нативное вставление товаров в SEO-статьи | 2 нед | средняя |
| **Webhook режим** | Триггер из Bitrix24 / Jira / Trello | 1 нед | низкая |
| **Yandex Direct / VK Ads** | Прямой апи для генерации промо-текстов | 2 нед | средняя |
| **Banner builder** (HTML/SVG) | Готовый файл баннера, не текст к нему | 2 нед | средняя |

Архитектура на это рассчитана — у нас уже есть выходной адаптер для HTML-вёрстки статей; остальные адаптеры пишутся по тому же шаблону.

## Чего НЕ можем (честно)

- ❌ **Модерация отзывов клиентов** (запрос Алины) — требует **отдельного контура** из-за PII по 152-ФЗ. Это другой проект, не текущий пилот.
- ❌ **Anthropic on-prem** — Anthropic не отдаёт веса. Митигация — self-hosted Qwen/DeepSeek для on-prem-чувствительных задач.
- ❌ **Real-time genrenerative ad personalization** (на уровне 1 ad per user) — это другой класс системы, не текущая задача.

## Вердикт по конкурентам

- **Все 7**: 🟢 — категориальная разница. Это не «у нас лучше», это «они в принципе про другое». Чат — окно для разговора. Мы — runtime для контента.
- **Custom GPT с Actions**: ближе всех, но Actions — DIY-конструктор. Каждая интеграция = своя API-обвязка, hosted у юзера, без валидатора, без git, без RUNBOOK.

## Честные оговорки

- **Большая часть интеграций — на roadmap**, не в текущем коде. Не давать иллюзию «всё готово завтра».
- **Каждая интеграция — отдельная работа** (1-2 недели), включается в стоимость поддержки или в отдельные пакеты.
- **Mindbox/Bitrix/SMS — каждая со своим API**: контракт нужно согласовать с владельцами систем у SimpleWine. Это не наша скорость, а скорость их IT.
- **SKU-интеграция (запрос Алины)** — реализуема, требует доступа к PIM и доку структуры. **2 недели** при наличии доступа.

## Главный аргумент для презентации

> «Конкуренты дают вам **окно браузера, в которое надо копировать бриф и из которого надо копировать результат**. Мы даём **API-точку**, в которую можно постучаться из Bitrix, из Mindbox, из Jira — и которая вернёт **готовый к публикации артефакт**: HTML-статью с картинками, JSON для push, кампанию для UniSender. У нас сегодня работает 4 интеграции (Anthropic + Gemini + DOCX + HTML-вёрстка). Любая следующая — 1-2 недели, потому что архитектура на это спроектирована.»

## Итог блока

**7/7 категорических побед.** Этот блок — про архитектурную **категорию** продукта. Чат-продукты не могут стать production runtime, не перестав быть чат-продуктами.

Это сильнейший аргумент против AlpinaGPT (он принципиально не интегрируется в Bitrix/CMS) и против Custom GPT (Actions — это DIY-конструктор, который надо собирать самому).
