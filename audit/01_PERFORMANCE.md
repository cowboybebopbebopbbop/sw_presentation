## Принцип блока

Меряем не «быстро ли отвечает чат», а «сколько готового, провалидированного контента уходит в продакшн за единицу времени без участия человека».

## Что реально есть

- **SEO-статья**: 7 часов копирайтера → ~2.5 часа (с генерацией + изображениями Gemini + готовой HTML-вёрсткой)
- **Баннер**: 2 часа → ~2 минуты до первого черновика
- **Multiformat**: 1 бриф → **65 текстов** (13 форматов × ~5 вариантов) одним прогоном с гарантией кросс-форматной консистентности
- **Запрос Алины**: 200 SEO-статей в месяц — архитектурно тянется (1 статья ~$0.37 → 200 шт ≈ 6800₽ токенов/мес)
- **Repair loop** до 3 попыток без участия человека
- **Tiered knowledge** (150-2500 vs 6300 токенов) — экономия времени и денег без потери качества

## Вердикт по конкурентам

- **ChatGPT/Claude (любая SKU)**: 🟢 — это интерфейсы для разговора, не для пакетной выдачи. 65 текстов одним прогоном там не получить, кросс-форматную консистентность скидки придётся ловить руками.
- **AlpinaGPT**: 🟢 — маршрутизатор, не пайплайн. Может «спросить лучшую модель», не может выдать готовый комплект форматов.
- **Yandex GPT / GigaChat**: 🟢 — то же самое: чат, не runtime.

## Честные оговорки

- **Batch SKU генерация** (1000 карточек товара одним прогоном) — пока не реализована, на roadmap. Если IT спросит «а 5000 карточек разом?» — честно сказать «Q2, архитектура поддерживает».
- **Параллельность ограничена rate-limit OpenRouter/Anthropic** — для 200 SEO/мес это не проблема, для 50 000 — нужен enterprise tier у Anthropic.
- **Скорость зависит от модели**: Opus 4.6/4.7 медленнее Sonnet/Haiku. Если критична скорость per-request, можно роутить простые шаги на Haiku/Gemini Flash.

## Главный аргумент для презентации

> «Конкуренты дают вам **окно для разговора с моделью**. Мы даём **production runtime**: один бриф → 65 готовых, проверенных, согласованных между собой текстов. Это разница между почтовым клиентом и почтовым сервером.»
