## Принцип блока

Меряем не «есть ли значок security», а **«где конкретно лежат данные брифа, кто к ним имеет доступ, что попадает за периметр компании и под какую юрисдикцию»**.

## Что реально есть (карта данных)

```
[Юзер вводит бриф в браузер]
    ↓
[localStorage браузера юзера]   ← API-ключ + последние брифы хранятся ЗДЕСЬ
    ↓
[Браузер делает HTTPS → OpenRouter (US/Canada)]
    ↓
[OpenRouter → Anthropic API (US)]
    ↓
[Готовый текст обратно в браузер]
    ↓
[Юзер копирует в Mindbox/CMS вручную]
```

**Важные факты:**
- **Своего бэкенда у нас нет** — нет нашей базы с брифами, нет нашего лога обращений
- **API-ключ хранится в `localStorage` в plain-text** — это известная слабость
- **За периметр компании уходит ТОЛЬКО текст брифа** (продуктовый контекст, цена, акция)
- **Обратно приходит ТОЛЬКО сгенерированный текст**
- **Никаких PII клиентов SimpleWine система не видит** — она работает с маркетинговыми брифами, не с базой клиентов

## Вердикт по конкурентам

- **ChatGPT web/Plus / Custom GPT / Projects**: 🟢 — наши данные за периметром, в US, без audit log, по умолчанию модель учится на промптах. Худший вариант для алкоголя.
- **ChatGPT Enterprise**: 🔴 **единственный честный 🔴 в этом блоке** — у них есть SOC 2, SAML SSO, audit log, no training. Наш ответ: «при тех же требованиях мы тоже это построим, **за тот же бюджет**, но с нашим валидатором и в рублях».
- **Claude.ai / Pro / Projects**: 🟡 — лучше OpenAI по приватности (no training by default), но всё равно US-юрисдикция, нет audit log, нет SSO.
- **AlpinaGPT**: 🟢 — формально РФ-юрлицо, но реально проксирует к OpenAI/Anthropic. Иллюзия суверенитета. Алина может это не знать.
- **Yandex GPT / GigaChat**: 🟢 — единственный реальный по РФ-юрисдикции конкурент. Качество текста ниже, но IT-директору это будет нравиться. **Наш ответ — multi-model: можно роутить чувствительные шаги на YaGPT, оставив Opus там, где нужно качество.**

## Honest gaps (надо признать ПЕРВЫМИ перед IT-директором)

| 🔴 Слабость | План фикса | Сроки |
